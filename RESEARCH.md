# 期待値研究の記録

最終更新: 2026-07-05

JRA単勝市場に対して期待値1.1以上のベットが可能かを検証する研究プログラムの記録。
サイト(mamahuhu)の予想モデルの根拠であり、「見送り宣言」路線の裏付けでもある。

## 方法論(全ラウンド共通)

- **Benter型2段階構造**: ファンダメンタルモデル f を市場確率 π と合成
  `p ∝ exp(α·ln f + β·ln π)`。市場のみのモデルは (α,β)=(0,1) でネスト
- **時系列3分割**(ランダム分割禁止): train 2024-01〜2025-06 / valid 2025-07〜12 /
  test 2026-01〜06。2023年分は特徴量のウォームアップ履歴
- **条件付きロジット**(レース内softmax)の尤度比検定 + train係数固定での
  valid/test再現性確認。trainで有意でもvalidで再現しなければ不採用
- **リーク防止**: 特徴量はすべて当該レース発走前に計算可能な情報のみ
- 注意: オッズは確定オッズ(前日オッズ運用では下振れしうる)

## 研究ラウンドと結果

| # | テーマ | スクリプト | 結果 |
|---|--------|-----------|------|
| 1 | 枠バイアス・脚質・日内更新 | research_features.py | 枠: β=+0.5だがn.s.(市場織り込み済み)。脚質×バイアス: 死に筋。日内: train有意もvalid再現せず |
| 2 | 王道12因子(直近成績・騎手残差・距離適性ほか) | research_fundamental.py | ファンダ単体は市場に大差で負ける(対数損失2.25 vs 1.91)。**αが負で3期間安定**(−0.08前後) → 当時は「市場が見た目の形を過大評価」と解釈 |
| 3 | スピード指数(日次トラックバリアント補正) | research_speed.py | ファンダ最強因子(2ΔLL=425)。**しかし合成でα→0**: 第2回の負αは「うちのモデルがタイム情報を欠いていた」だけ。市場はタイム指数を完全に織り込み済み |
| 4 | trip handicapping(ペース逆風・バイアス日和見補正・展開適性) | research_trip.py | ペース逆風は実在する因子(train有意)だが、合成でα≈−0.01でn.s.。**市場超え完全消滅** |
| 5 | 日内更新の精緻化(急変日限定・距離帯・開催後半) | research_intraday.py | **全滅**。急変日はtrainですらn.s.、効果はむしろ安定日に集中=仮説と真逆。第1回のtrain有意は過学習と確定 |

αの推移(ファンダを改善するほど市場との差分が消える):
12因子 −0.08 → +speed −0.03 → +trip −0.01。市場の価格 ≈ 手元データから作れる最良モデル。

## 確定した結論

1. **JRA単勝市場は、過去データから計算可能な全ての公開情報に対して効率的**
   (着順・タイム・ペース・展開・騎手・距離適性・枠/脚質バイアス履歴)
2. 期待値1.1に必要な優位(市場比+26%以上の確率評価)に対し、検出された効果は
   最大でも対数損失改善0.002未満 = 控除率の壁(約0.14)の1/100
3. 唯一一貫して残るのは β(市場)≈1.03〜1.08 の弱い本命バイアス
   (市場確率はややシャープにすべき)。ただし搾取可能な水準ではない
4. パリミュチュエル方式のため「早買いで値上がり益」は構造的に存在しない。
   残る勝ち筋は市場の**反応速度・構造**を突くものだけ

→ サイトの「正直な見送り宣言 + 注目馬(相対シグナル)」路線はこの結果の直接の帰結。

## 検証待ちの仮説

1. **オッズ変動(スマートマネー追随)** — 直前に売れた馬は確定オッズでもなお
   勝ちすぎるか。英国固定オッズ市場では実証あり(steamers)、パリミュチュエルでは未知。
   条件付きロジットに log(確定/前日オッズ比) を追加して検定。
   データ要件: 週約70R × 1〜3ヶ月(2026年8〜10月頃に第1回)
2. **柵替わり×バイアスリセット** — A→B替わり週の内枠復活を市場が正しく
   価格付けしているか。course_setting蓄積後(数ヶ月)
3. **複雑券種の歪み** — 単勝確率モデル→Harville式(+Henery/Stern補正)で
   三連系を価格付けし、実払戻と突合。歪みが乗算で増幅されるかの検証。
   payouts蓄積後。過去分バックフィルで前倒し可能
4. (未着手) コメント/映像ベースの不利検出 — データソースなし

## 実行方法

```bash
# 研究スクリプト(いずれも keiba.db / site.db を読むだけ)
python scraper/research_features.py    # 第1回
python scraper/research_fundamental.py # 第2回
python scraper/research_speed.py       # 第3回
python scraper/research_trip.py        # 第4回
python scraper/research_intraday.py    # 第5回

# 前日オッズ・順方向予想
python scraper/odds.py --date 20260711
python scraper/predict.py 小倉 --date 20260711 --forward
python scraper/rebuild_site.py --forward 20260711   # 発走前予想をsite.dbへ

# 払戻の過去分バックフィル(お試し)
python scraper/backfill.py --payouts-only --from 20240106 --to 20240107 --limit 5
```

運用サイクル(全自動): 金18時 notable更新+翌土曜オッズ+土曜分の順方向予想を
サイト公開(土曜朝までに土曜の期待値分析が出る) → 金土20時 前日ポスト(注目馬
実名入り) → 土日朝8時 当日オッズを再取得して順方向予想を更新(オッズ変動を
反映。旧スナップショットは forward_odds_history に退避) → 土日18時
scrape+rebuild+翌日オッズ+翌日分の順方向予想をサイト公開(日曜朝までに日曜分
が出る) → 火18時 rawdb+払戻+rebuild → 火20時 検証ポスト。
発走前の順方向予想(pred_races.forward=1、snapped_atがオッズ取得時点)は、
結果確定後の rebuild で確定オッズ版(forward=0)に丸ごと置き換わる。

詳細な設計判断は `.claude/bias_analysis_spec.md` を参照。
